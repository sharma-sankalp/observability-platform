# Observability Overview

## Purpose

Observability is the ability to understand the internal state of a system by collecting and analyzing telemetry data. It enables engineers to detect issues, investigate failures, and improve application reliability.

---

# The Three Pillars

## Metrics

Metrics are numerical measurements collected over time.

Examples:

- CPU Usage
- Memory Usage
- Request Rate
- Error Rate
- Latency

Prometheus is commonly used to collect and store metrics.

---

## Logs

Logs are timestamped records of events generated by applications and infrastructure.

Examples include:

- Application logs
- System logs
- Kubernetes logs
- Audit logs

Loki provides centralized log aggregation.

---

## Traces

Distributed traces follow a request across multiple services.

Tracing helps identify:

- Slow requests
- Service dependencies
- Root causes
- Performance bottlenecks

OpenTelemetry provides vendor-neutral instrumentation.

---

# Benefits

A well-designed observability platform enables:

- Faster incident detection
- Improved troubleshooting
- Performance optimization
- Capacity planning
- Service reliability
- Better operational visibility
